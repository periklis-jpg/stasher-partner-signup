# Deploy Lambda NOW (required for Pending affiliates)

The signup **frontend** auto-deploys from GitHub via Amplify.

The **Lambda backend does NOT**. Until you update it, email-based enrollment recovery cannot run and some affiliates will still land in **Archived**.

## What I verified on the live API (May 2026)

| Call | Result |
|------|--------|
| `create_affiliate_only` + `finalize_affiliate` | Works → `approved: null` (Pending) when both return 200 |
| `complete_enrollment` / finalize-by-email | **Not deployed** → falls through to legacy and fails |
| Legacy / single-shot | Works sometimes, **502 timeouts** leave affiliates Archived |
| Root cause of Archived | Affiliate created in Tapfiliate, **program enrollment never completes** |

## Deploy steps (5 minutes)

1. Open **AWS Console** → **Lambda** → region **eu-north-1**
2. Find the function behind `wpnp6ab1ge.execute-api.eu-north-1.amazonaws.com/prod/create-affiliate`
3. **Code** tab → replace `index.js` with:
   `amplify/backend/function/createAffiliate/src/index.js` from this repo
4. **Deploy**
5. **Configuration** → **General configuration** → set **Timeout** to **30 seconds** (not 3s — 502s cause Archived affiliates)
6. Verify env var `TAPFILIATE_API_KEY` is set

Or upload `createAffiliate-lambda.zip` from the repo root (generated by `npm run zip-lambda`).

## After deploy — quick test

```bash
curl -s -X POST 'https://wpnp6ab1ge.execute-api.eu-north-1.amazonaws.com/prod/create-affiliate' \
  -H 'Content-Type: application/json' \
  -d '{"mode":"finalize_affiliate","email":"YOUR_TEST_EMAIL","program":"stasher-affiliate-program","program_currency":"GBP"}'
```

Should return `"approved": null`, not `"Missing affiliate_id"`.

## Archived affiliates already created

Delete them in Tapfiliate or manually add them to the correct program as Pending. The signup fix prevents **new** ones; it does not retroactively repair old ones unless you enroll them manually.
